Wireguard GNOME clinet profile setup

Table of content

  • Adding new configuration based on file
  • Adding manually
    • Setting IP

Adding Wireguard client profiles is now possible directly through GNOME GUI interface, without need to tinker with the CLI.

Adding new configuration based on file

After downloading peer confing generated by Wireguard server we can easily import it to VPN settings.

[Interface]
Address = 10.0.0.3
PrivateKey = <my private key>
ListenPort = 51820
DNS = 127.0.0.11,8.8.8.8,1.1.1.1

[Peer]
PublicKey = <my public key>
PresharedKey = <preshered key>
Endpoint = <wireguard ip>:51820
AllowedIPs = 0.0.0.0/0

Navigate to Settings -> Network -> VPN - click on plus sign.

You can import from file, it will automatically add Wireguard configuration.

Adding manually

In case you would like to configure the Wireguard connection manually, select “Wireguard” box and navigate to “Wireguard” tab.

Configure:

  • Connection Name: How you would like to see the VPN connection in the menu
  • Interface Name: Usually name of the peer
  • Private Key: Copy your private key from config
  • Listen Port: Port on which Wireguard will use for transacions on your device usually 51820
  • fwmark: Set to 0
  • MTU: Set to 0 bytes

Select “Add peer routes”.

Click + sign next to the peers, and configure the connection:

  • Endpoint: IP or hostname of your Wireguard server
  • Public Key: Copy public key from the config
  • Preshared Key: Copy preshared key from the config
  • Allowed IPs: Copy allowed IPs from the config
  • Persistent Keep Alive: In case you are using this set to proper value from the config

Setting IP

In case you are not using DHCP navigate to IPv4 tab, and select IPv4 method to “Manual”.

Configure in “Addresses” table:

  • Address: From config - 10.0.0.3
  • Netmask: Usually 255.255.255.255

In the DNS use 127.0.0.11, 8.8.8.8, 1.1.1.1. First DNS is Docker’s DNS which allow us to use names of services as IPs internally in the Wireguard container.
